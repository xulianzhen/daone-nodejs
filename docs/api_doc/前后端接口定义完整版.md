# Daone 前后端接口定义 v1

> 本文替代原《前后端接口约定》。一期接口保持 REST 风格，不为未来功能预建空接口。

## 1. 通用约定

### 1.1 基础信息

- Base URL：`/api/v1`
- Content-Type：`application/json`
- 时间：ISO 8601，统一返回 UTC，例如 `2026-06-14T12:30:00Z`
- ID：统一使用字符串
- 金额：整数分，例如 `599900` 表示 5999 元
- 积分：整数
- 登录请求头：`Authorization: Bearer <token>`
- 幂等写操作请求头：`Idempotency-Key: <uuid>`

### 1.2 统一响应

成功：

```json
{
  "code": "OK",
  "message": "success",
  "data": {},
  "traceId": "01J..."
}
```

失败：

```json
{
  "code": "POINTS_NOT_ENOUGH",
  "message": "积分不足",
  "data": null,
  "traceId": "01J..."
}
```

HTTP 状态码：

| 状态码 | 含义 |
|---:|---|
| 200 | 查询或更新成功 |
| 201 | 创建成功 |
| 400 | 参数错误 |
| 401 | 未登录或 token 失效 |
| 403 | 无权限或账号禁用 |
| 404 | 资源不存在 |
| 409 | 状态或版本冲突 |
| 429 | 请求过于频繁 |
| 500 | 系统错误 |
| 502 | 外部模型或支付服务异常 |

### 1.3 分页

请求：

```text
page=1&pageSize=20
```

响应：

```json
{
  "items": [],
  "page": 1,
  "pageSize": 20,
  "total": 0
}
```

`page` 从 1 开始，`pageSize` 默认 20，最大 100。

## 2. 认证

### 2.1 发送短信验证码

`POST /auth/sms-codes`

```json
{
  "phone": "13800138000",
  "scene": "LOGIN"
}
```

响应：

```json
{
  "retryAfterSeconds": 60
}
```

`scene` 一期仅支持 `LOGIN`。验证码固定返回布尔值没有意义，发送失败直接返回错误码。

### 2.2 手机验证码登录

`POST /auth/sms-login`

```json
{
  "phone": "13800138000",
  "code": "123456"
}
```

响应：

```json
{
  "token": "dn_xxx",
  "expiresInSeconds": 604800,
  "user": {
    "id": "193...",
    "nickname": "Daone用户",
    "avatarUrl": null
  }
}
```

### 2.3 创建微信扫码登录

`POST /auth/wechat/qr-sessions`

响应：

```json
{
  "ticket": "wxt_01J...",
  "authorizeUrl": "https://open.weixin.qq.com/...",
  "expiresAt": "2026-06-14T12:35:00Z"
}
```

前端根据 `authorizeUrl` 生成或展示二维码。

### 2.4 查询微信扫码状态

`GET /auth/wechat/qr-sessions/{ticket}`

未扫码：

```json
{
  "status": "WAITING"
}
```

已扫码成功：

```json
{
  "status": "CONFIRMED",
  "token": "dn_xxx",
  "expiresInSeconds": 604800
}
```

状态枚举：`WAITING`、`SCANNED`、`CONFIRMED`、`EXPIRED`。

### 2.5 退出登录

`POST /auth/logout`

响应：

```json
{}
```

## 3. 用户与账户

### 3.1 获取当前用户

`GET /users/me`

```json
{
  "id": "193...",
  "nickname": "李阳",
  "avatarUrl": "https://...",
  "phoneMasked": "189****2675",
  "email": null,
  "gender": "UNKNOWN",
  "birthday": null,
  "subscription": {
    "planCode": "TEAM",
    "planName": "团队协作版",
    "expireAt": "2027-06-14T00:00:00Z",
    "autoRenew": false
  },
  "points": {
    "available": 12003,
    "frozen": 0,
    "grantedTotal": 12003
  }
}
```

`gender`：`MALE`、`FEMALE`、`UNKNOWN`。

### 3.2 修改个人资料

`PATCH /users/me`

只传需要修改的字段：

```json
{
  "nickname": "新昵称",
  "avatarAssetId": "193...",
  "email": "user@example.com",
  "gender": "UNKNOWN",
  "birthday": "1995-01-01"
}
```

手机号和微信绑定不通过此接口修改。

一期不支持密码登录，因此原型“登录密码”字段应隐藏。手机号修改必须增加短信二次验证，不通过资料编辑接口直接修改；一期可先置灰。

### 3.3 查询积分账户

`GET /points/account`

```json
{
  "available": 12003,
  "frozen": 20,
  "grantedTotal": 12003
}
```

### 3.4 查询积分流水

`GET /points/ledger?page=1&pageSize=20&direction=DECREASE`

```json
{
  "items": [
    {
      "id": "193...",
      "action": "CONSUME",
      "amount": -20,
      "balanceAfter": 12003,
      "description": "图片生成",
      "bizType": "GENERATION_TASK",
      "bizId": "193...",
      "createdAt": "2026-06-14T12:30:00Z"
    }
  ],
  "page": 1,
  "pageSize": 20,
  "total": 1
}
```

`action`：`GRANT`、`RECHARGE`、`FREEZE`、`CONSUME`、`REFUND`、`ADJUST`。

`direction`：`ALL`、`INCREASE`、`DECREASE`，对应原型中的全部、增加、减少。

### 3.5 查询积分流水详情

`GET /points/ledger/{ledgerId}`

返回本次积分变化、关联生成任务、提示词摘要、模型、失败原因和发生时间。用于原型积分日志中的“详情”按钮。

`PATCH /projects/{projectId}`

```json
{
  "title": "夏季商品海报",
  "coverAssetId": "193..."

```

### 4.5 删除项目

`DELETE /projects/{projectId}`

响应 HTTP 204，无响应体。删除为逻辑删除，不立即删除 OSS 文件。

### 4.6 获取当前画布

`GET /projects/{projectId}/canvas`

```json
{
  "projectId": "193...",
  "revision": 8,
  "canvasData": {
    "schemaVersion": 1,
    "nodes": [],
    "edges": [],
    "groups": [],
    "viewport": {"x": 0, "y": 0, "zoom": 1}
  },
  "updatedAt": "2026-06-14T12:30:00Z"
}
```

`canvasData` 使用 JSON 对象，不使用二次编码的 JSON 字符串。

### 4.7 保存当前画布

`PUT /projects/{projectId}/canvas`

```json
{
  "revision": 8,
  "saveType": "AUTO",
  "canvasData": {
    "schemaVersion": 1,
    "nodes": [],
    "edges": [],
    "groups": [],
    "viewport": {"x": 0, "y": 0, "zoom": 1}
  }
}
```

响应：

```json
{
  "revision": 9,
  "savedAt": "2026-06-14T12:31:00Z"
}
```

`saveType`：`AUTO`、`MANUAL`。`MANUAL` 会额外创建历史版本。

revision 冲突返回 HTTP 409：

```json
{
  "code": "CANVAS_REVISION_CONFLICT",
  "message": "画布已在其他页面更新",
  "data": {"latestRevision": 10},
  "traceId": "01J..."
}
```

### 4.8 历史版本列表

`GET /projects/{projectId}/versions?page=1&pageSize=20`

```json
{
  "items": [
    {
      "id": "193...",
      "versionNo": 3,
      "createdAt": "2026-06-14T12:30:00Z"
    }
  ],
  "page": 1,
  "pageSize": 20,
  "total": 1
}
```

### 4.9 获取历史版本

`GET /projects/{projectId}/versions/{versionId}`

返回 `versionNo`、`canvasData` 和 `createdAt`。

### 4.10 恢复历史版本

`POST /projects/{projectId}/versions/{versionId}/restore`

```json
{
  "revision": 10
}
```

响应返回新的 `revision`，恢复动作同时生成一份手动版本。

## 5. 素材

### 5.1 获取上传凭证

`POST /assets/upload-tickets`

```json
{
  "projectId": "193...",
  "fileName": "product.png",
  "contentType": "image/png",
  "fileSize": 1024000
}
```

响应：

```json
{
  "uploadTicket": "upt_01J...",
  "uploadMethod": "PUT",
  "uploadUrl": "https://bucket.oss-cn-hangzhou.aliyuncs.com/user/...?signature=...",
  "objectKey": "user/193/2026/06/uuid.png",
  "formFields": {
    "Content-Type": "image/png"
  },
  "expiresAt": "2026-06-14T12:40:00Z"
}
```

`objectKey` 由后端生成，前端不得自定义用户目录。测试和生产环境按照
`uploadMethod=PUT` 上传文件，并把 `formFields` 作为请求头；本地环境返回
`uploadMethod=POST`，使用 multipart 表单上传。

### 5.2 确认上传完成

`POST /assets`

```json
{
  "uploadTicket": "upt_01J...",
  "projectId": "193...",
  "fileSize": 1024000
}
```

响应：

```json
{
  "id": "193...",
  "type": "IMAGE",
  "fileName": "product.png",
  "fileSize": 1024000,
  "status": "REVIEWING",
  "createdAt": "2026-06-14T12:30:00Z"
}
```

### 5.3 素材列表

`GET /assets?scope=MINE&projectId=&type=IMAGE&source=UPLOAD&keyword=&page=1&pageSize=20`

响应项：

```json
{
  "id": "193...",
  "type": "IMAGE",
  "source": "UPLOAD",
  "fileName": "product.png",
  "previewUrl": "https://signed-url...",
  "width": 1024,
  "height": 1024,
  "durationSeconds": null,
  "status": "AVAILABLE",
  "favorited": false,
  "createdAt": "2026-06-14T12:30:00Z"
}
```

`type`：`IMAGE`、`VIDEO`、`TEXT`。  
`source`：`UPLOAD`、`GENERATED`、`TEMPLATE`。  
`status`：`REVIEWING`、`AVAILABLE`、`REJECTED`、`DELETED`。

`scope` 对应前端原型：

| scope | 页面 |
|---|---|
| `RECOMMENDED` | 智能推荐 |
| `CENTER` | 素材中心 |
| `MINE` | 我的素材 |
| `FAVORITE` | 我的收藏 |
| `FILES` | 我的文件，包含上传素材和生成结果 |

### 5.4 素材详情

`GET /assets/{assetId}`

返回素材元数据和短时有效的 `previewUrl`、`downloadUrl`。

### 5.5 删除素材

`DELETE /assets/{assetId}`

响应 HTTP 204。已被画布引用时仍可逻辑删除，但历史项目中的对象在保留期内不立即物理清理。

### 5.6 收藏素材

`PUT /assets/{assetId}/favorite`

响应：

```json
{"favorited": true}
```

### 5.7 取消收藏

`DELETE /assets/{assetId}/favorite`

响应 HTTP 204。

## 6. AI 能力与生成任务

### 6.1 获取可用能力

`GET /ai/capabilities?taskType=IMAGE`

```json
{
  "items": [
    {
      "code": "IMAGE_GENERAL_V1",
      "name": "通用图片生成",
      "taskType": "IMAGE",
      "estimatedPoints": 20,
      "parameterSchema": {
        "aspectRatio": ["1:1", "3:4", "4:3", "16:9"],
        "resolution": ["1K", "2K"],
        "count": {"min": 1, "max": 4}
      }
    }
  ]
}
```

该接口替代含义不清的通用“字典接口”。前端只展示后端返回且已启用的模型和参数。

### 6.2 预估积分

`POST /ai/point-estimates`

```json
{
  "capabilityCode": "IMAGE_GENERAL_V1",
  "parameters": {
    "resolution": "2K",
    "count": 2
  }
}
```

响应：

```json
{
  "estimatedPoints": 40
}
```

### 6.3 创建生成任务

`POST /generation-tasks`

请求头必须携带 `Idempotency-Key`。

```json
{
  "projectId": "193...",
  "nodeId": "node-12",
  "taskType": "IMAGE",
  "capabilityCode": "IMAGE_GENERAL_V1",
  "prompt": "白色背景的夏季运动鞋商品图",
  "referenceAssetIds": ["193..."],
  "parameters": {
    "aspectRatio": "1:1",
    "resolution": "2K",
    "count": 2
  }
}
```

响应 HTTP 201：

```json
{
  "id": "193...",
  "status": "QUEUED",
  "estimatedPoints": 40,
  "createdAt": "2026-06-14T12:30:00Z"
}
```

文本、图片、视频均使用此接口，差异由 `taskType`、`capabilityCode` 和 `parameters` 表达，不为每个模型增加独立接口。

### 6.4 查询任务

`GET /generation-tasks/{taskId}`

```json
{
  "id": "193...",
  "projectId": "193...",
  "nodeId": "node-12",
  "taskType": "IMAGE",
  "status": "RUNNING",
  "progress": 45,
  "estimatedPoints": 40,
  "actualPoints": null,
  "results": [],
  "error": null,
  "createdAt": "2026-06-14T12:30:00Z",
  "updatedAt": "2026-06-14T12:31:00Z"
}
```

成功时：

```json
{
  "status": "SUCCEEDED",
  "progress": 100,
  "actualPoints": 40,
  "results": [
    {
      "assetId": "193...",
      "type": "IMAGE",
      "previewUrl": "https://signed-url..."
    }
  ]
}
```

前端建议运行中每 2 秒轮询一次，页面进入后台后降低频率。

### 6.5 任务列表

`GET /generation-tasks?projectId=&status=&taskType=&keyword=&dateFrom=&dateTo=&page=1&pageSize=20`

用于恢复运行中任务，以及支持画布“历史记录”中的搜索、图片/视频/文字/自定义分类和日期筛选。列表项应返回 `promptSummary`、`taskType`、`status`、`createdAt` 和结果缩略图。

### 6.6 取消任务

`POST /generation-tasks/{taskId}/cancel`

只有 `QUEUED` 或供应商支持取消的 `RUNNING` 任务可取消。无法取消返回 `TASK_CANNOT_CANCEL`。

### 6.7 提示词翻译

`POST /ai/prompt-translations`

```json
{
  "text": "白色背景的运动鞋",
  "targetLanguage": "EN"
}
```

```json
{
  "translatedText": "Sports shoes on a white background"
}
```

是否消耗积分由后台能力配置决定。

### 6.8 创建 AI 对话

`POST /chat-sessions`

```json
{
  "projectId": "193...",
  "title": "New Chat"
}
```

响应返回 `id`、`title` 和 `createdAt`。

### 6.9 AI 对话列表

`GET /chat-sessions?projectId=193...&page=1&pageSize=20`

用于编辑器右侧“历史记录”，返回会话标题、最后一条消息摘要和更新时间。

### 6.10 对话消息列表

`GET /chat-sessions/{sessionId}/messages?page=1&pageSize=50`

消息项：

```json
{
  "id": "193...",
  "role": "USER",
  "content": "生成一张白底运动鞋商品图",
  "attachmentAssets": [],
  "generationTaskIds": [],
  "createdAt": "2026-06-14T12:30:00Z"
}
```

`role`：`USER`、`ASSISTANT`。

### 6.11 发送对话消息

`POST /chat-sessions/{sessionId}/messages`

```json
{
  "content": "生成一张白底运动鞋商品图",
  "skillCode": "ECOMMERCE_IMAGE",
  "modelCode": "IMAGE_GENERAL_V1",
  "attachmentAssetIds": ["193..."]
}
```

响应返回助手消息和可能创建的 `generationTaskIds`。实际生成结果仍通过生成任务接口轮询。

### 6.12 删除 AI 对话

`DELETE /chat-sessions/{sessionId}`

响应 HTTP 204，不删除该会话已经生成的素材。

### 6.13 获取可用技能

`GET /ai/skills?scene=CHAT`

```json
{
  "items": [
    {
      "code": "ECOMMERCE_IMAGE",
      "name": "电商商品图",
      "description": "根据商品与提示词生成电商图片",
      "supportedTaskTypes": ["IMAGE"],
      "enabled": true
    }
  ]
}
```

右侧面板的 Models 复用 `/ai/capabilities`，Skills 使用本接口。原型中的“技能创建”一期不开放给普通用户。

## 8. 套餐、订单与支付

### 8.1 套餐列表

`GET /plans?orderType=PLAN`

```json
{
  "items": [
    {
      "code": "TEAM_YEAR",
      "name": "团队协作版",
      "cycle": "YEAR",
      "priceFen": 599900,
      "originalPriceFen": 799900,
      "grantPoints": 12000,
      "benefits": [
        "3 人成员协作",
        "150G 存储空间"
      ]
    }
  ]
}
```

`cycle`：`MONTH`、`YEAR`、`TWO_YEARS`。

### 8.2 创建订单

`POST /orders`

请求头必须携带 `Idempotency-Key`。

```json
{
  "orderType": "PLAN",
  "productCode": "TEAM_YEAR"
}
```

响应：

```json
{
  "orderNo": "DN20260614...",
  "amountFen": 599900,
  "status": "PENDING",
  "expireAt": "2026-06-14T12:45:00Z"
}
```

`orderType`：`PLAN`、`POINT_RECHARGE`。

### 8.3 创建支付

`POST /orders/{orderNo}/payments`

```json
{
  "payType": "WECHAT"
}
```

微信 Native 支付响应：

```json
{
  "payType": "WECHAT",
  "qrCodeContent": "weixin://wxpay/...",
  "expireAt": "2026-06-14T12:45:00Z"
}
```

支付宝响应可返回：

```json
{
  "payType": "ALIPAY",
  "redirectUrl": "https://..."
}
```

### 8.4 查询订单

`GET /orders/{orderNo}`

```json
{
  "orderNo": "DN20260614...",
  "orderType": "PLAN",
  "productName": "团队协作版",
  "amountFen": 599900,
  "payType": "WECHAT",
  "status": "PAID",
  "paidAt": "2026-06-14T12:35:00Z",
  "createdAt": "2026-06-14T12:30:00Z"
}
```

订单状态：`PENDING`、`PAYING`、`PAID`、`CLOSED`。

### 8.5 订单列表

`GET /orders?page=1&pageSize=20&status=PAID`

返回分页订单。前端轮询订单只用于刷新展示，支付成功最终以服务端回调为准。

原型账单页存在“开票”按钮，但 PRD 未要求电子发票，一期不提供开票接口，前端应先隐藏或置灰。

### 8.6 取消自动续费

`POST /subscriptions/cancel-auto-renew`

关闭当前生效订阅的自动续费，不影响本期套餐继续使用。

本地联调环境提供 `POST /orders/{orderNo}/mock-paid` 模拟支付成功。该接口只在
`local` Profile 生效，生产环境不会注册。

微信和支付宝服务端通知地址分别为：

- `POST /payments/WECHAT/notify`
- `POST /payments/ALIPAY/notify`

通知接口不要求用户登录，但必须由支付客户端完成验签、金额和币种校验，
前端不得主动调用。

| PATCH | `/users/{userId}/status` | 启用或禁用用户 |

## 13. 当前原型需要同步的点

1. 左侧“项目”入口实际展示素材中心，应改名为“素材”或新增真正的“我的项目”页面。
2. 素材页五个入口已通过 `scope` 参数覆盖；“我的收藏”需要接收藏接口。
3. 画布历史记录需要使用生成任务的 `keyword/taskType/dateFrom/dateTo` 筛选。
4. 右侧 AI 对话需要对接会话和消息接口，生成结果仍绑定 `generationTaskId`。
5. 原型“编辑资料”中的密码字段应删除；系统一期没有密码登录。
6. 手机号不能直接修改，一期应置灰；后续如开放必须使用短信二次验证。
7. 账单“开票”、消息通知、申请试用一期未纳入 PRD，应隐藏或展示“暂未开放”，不新增后端模块。
8. “充值”当前打开的是会员套餐，不是独立积分商品。若一期没有单独积分充值，应将按钮改为“升级套餐”。
9. 套餐文案包含团队成员协作、画布协作和无限并发，但一期不实现这些能力，正式上线前必须调整文案。
10. 画布自动保存必须携带最新 `revision`，收到 409 时先拉取最新画布，不应静默覆盖。
11. 前端不能长期保存 OSS 永久 URL；素材展示地址由后端返回短时签名 URL。
12. 导出一期只支持下载单个生成素材；批量导出或画布 ZIP 需另立需求。
